# Database Design MVP

## Игра “Овощные Войны”

---

# 1. players

## Назначение

Хранит основной профиль игрока, который заходит в игру через Telegram.

```sql
create table players (
    id uuid primary key,
    telegram_id bigint not null unique,
    username varchar(100),
    first_name varchar(100),
    avatar_code varchar(50),
    created_at timestamptz not null default now(),
    updated_at timestamptz not null default now()
);
```

| Поле        | Назначение                                 |
| ----------- | ------------------------------------------ |
| id          | Внутренний идентификатор игрока            |
| telegram_id | ID пользователя Telegram                   |
| username    | Username из Telegram                       |
| first_name  | Имя пользователя из Telegram               |
| avatar_code | Код выбранного аватара / Хранителя огорода |
| created_at  | Дата создания профиля                      |
| updated_at  | Дата последнего обновления                 |

# 2. player_currencies

## Назначение

Хранит баланс валют игрока.

```sql
create table player_currencies (
    player_id uuid primary key references players(id),
    seeds int not null default 0,
    stars_balance int not null default 0,
    tournament_tokens int not null default 0,
    updated_at timestamptz not null default now()
);
```

| Поле              | Назначение                         |
| ----------------- | ---------------------------------- |
| player_id         | Игрок, которому принадлежит баланс |
| seeds             | Основная игровая валюта — семечки  |
| stars_balance     | Баланс Telegram Stars внутри игры  |
| tournament_tokens | Турнирные жетоны                   |
| updated_at        | Дата последнего изменения баланса  |

# 3. gardens

## Назначение

Хранит огород игрока.

```sql
create table gardens (
    id uuid primary key,
    player_id uuid not null unique references players(id),
    name varchar(100),
    created_at timestamptz not null default now(),
    updated_at timestamptz not null default now()
);
```

| Поле       | Назначение                |
| ---------- | ------------------------- |
| id         | Идентификатор огорода     |
| player_id  | Владелец огорода          |
| name       | Название огорода          |
| created_at | Дата создания огорода     |
| updated_at | Дата последнего изменения |

# 4. garden_plots

## Назначение

Хранит грядки внутри огорода.

```sql
create table garden_plots (
    id uuid primary key,
    garden_id uuid not null references gardens(id),
    plot_index int not null,
    is_unlocked boolean not null default true,
    is_greenhouse boolean not null default false,
    greenhouse_level int not null default 0,
    created_at timestamptz not null default now(),
    updated_at timestamptz not null default now(),

    unique(garden_id, plot_index)
);
```

| Поле             | Назначение                          |
| ---------------- | ----------------------------------- |
| id               | Идентификатор грядки                |
| garden_id        | Огород, которому принадлежит грядка |
| plot_index       | Номер грядки                        |
| is_unlocked      | Открыта ли грядка для использования |
| is_greenhouse    | Есть ли на грядке теплица           |
| greenhouse_level | Уровень теплицы                     |
| created_at       | Дата создания грядки                |
| updated_at       | Дата последнего изменения           |

## Примечание

Теплица ускоряет рост культуры на этой грядке, но не усиливает персонажа в бою.

# 5. crop_types

## Назначение

Справочник всех доступных культур: картошка, огурец, помидор, кукуруза и т.д.

```sql
create table crop_types (
    id uuid primary key,
    code varchar(50) not null unique,
    name varchar(100) not null,
    role varchar(50) not null,
    base_hp int not null,
    base_attack int not null,
    base_defense int not null,
    attack_range varchar(50) not null,
    attack_speed_ms int not null,
    ability_code varchar(50) not null references abilities(code),
    grow_time_seconds int not null,
    rarity varchar(50) not null default 'common',
    image_url text,
    created_at timestamptz not null default now()
);
```

| Поле              | Назначение                  |
| ----------------- | --------------------------- |
| id                | Идентификатор типа культуры |
| code              | Внутренний код культуры     |
| name              | Название культуры           |
| role              | Роль в бою                  |
| base_hp           | Базовое здоровье            |
| base_attack       | Базовая атака               |
| base_defense      | Базовая защита              |
| attack_range      | Тип дистанции атаки         |
| attack_speed_ms   | Частота обычной атаки       |
| ability_code      | Код суперспособности        |
| grow_time_seconds | Время роста культуры        |
| rarity            | Редкость культуры           |
| image_url         | Ссылка на изображение       |
| created_at        | Дата создания записи        |

## Примеры role

```text
tank
fighter
ranged
support
assassin
```

## Примеры attack_range

```text
melee
ranged
support
```

# 6. abilities

## Назначение

Справочник суперспособностей персонажей.

```sql
create table abilities (
    code varchar(50) primary key,
    name varchar(100) not null,
    description text,
    cooldown_seconds int not null,
    effect_type varchar(50) not null,
    effect_value int not null,
    target_type varchar(50) not null,
    created_at timestamptz not null default now()
);
```

| Поле             | Назначение                      |
| ---------------- | ------------------------------- |
| code             | Уникальный код способности      |
| name             | Название способности            |
| description      | Описание способности            |
| cooldown_seconds | Время перезарядки               |
| effect_type      | Тип эффекта                     |
| effect_value     | Значение эффекта                |
| target_type      | На кого применяется способность |
| created_at       | Дата создания записи            |

## Примеры effect_type

```text
damage
heal
shield
stun
dodge
buff_attack
```

## Примеры target_type

```text
enemy_single
enemy_all
ally_single
ally_all
self
```

# 7. player_crops

## Назначение

Хранит конкретные культуры, которые принадлежат игроку.

```sql
create table player_crops (
    id uuid primary key,
    player_id uuid not null references players(id),
    crop_type_id uuid not null references crop_types(id),
    plot_id uuid references garden_plots(id),
    status varchar(50) not null,
    planted_at timestamptz,
    ready_at timestamptz,
    activated_at timestamptz,
    created_at timestamptz not null default now(),
    updated_at timestamptz not null default now()
);
```

| Поле         | Назначение                         |
| ------------ | ---------------------------------- |
| id           | Идентификатор культуры игрока      |
| player_id    | Владелец культуры                  |
| crop_type_id | Тип культуры                       |
| plot_id      | Грядка, на которой культура растет |
| status       | Состояние культуры                 |
| planted_at   | Когда была посажена                |
| ready_at     | Когда будет готова                 |
| activated_at | Когда стала доступна для боев      |
| created_at   | Когда культура появилась у игрока  |
| updated_at   | Дата последнего изменения          |

## Статусы

```text
seed
growing
ready
active
```

# 8. player_team

## Назначение

Хранит текущую команду игрока для боя.

```sql
create table player_team (
    player_id uuid not null references players(id),
    slot int not null,
    player_crop_id uuid not null references player_crops(id),
    updated_at timestamptz not null default now(),

    primary key(player_id, slot)
);
```

| Поле           | Назначение                      |
| -------------- | ------------------------------- |
| player_id      | Игрок                           |
| slot           | Номер позиции в команде         |
| player_crop_id | Культура, выбранная в этот слот |
| updated_at     | Дата изменения команды          |

## Примечание

Для MVP используется формат 3 на 3.

```text
slot = 1
slot = 2
slot = 3
```

# 9. battles

## Назначение

Хранит общую информацию о бое.

```sql
create table battles (
    id uuid primary key,
    battle_type varchar(50) not null,
    status varchar(50) not null,
    player1_id uuid not null references players(id),
    player2_id uuid not null references players(id),
    winner_player_id uuid references players(id),
    started_at timestamptz,
    finished_at timestamptz,
    created_at timestamptz not null default now()
);
```

| Поле             | Назначение           |
| ---------------- | -------------------- |
| id               | Идентификатор боя    |
| battle_type      | Тип боя              |
| status           | Статус боя           |
| player1_id       | Первый игрок         |
| player2_id       | Второй игрок         |
| winner_player_id | Победитель боя       |
| started_at       | Время начала боя     |
| finished_at      | Время завершения боя |
| created_at       | Дата создания боя    |

## battle_type

```text
normal
tournament
```

## status

```text
created
waiting
in_progress
finished
cancelled
```

# 10. battle_units

## Назначение

Хранит состав участников боя и финальное состояние каждого персонажа.

```sql
create table battle_units (
    id uuid primary key,
    battle_id uuid not null references battles(id),
    player_id uuid not null references players(id),
    player_crop_id uuid not null references player_crops(id),
    crop_type_id uuid not null references crop_types(id),
    side varchar(10) not null,
    slot int not null,
    start_hp int not null,
    end_hp int not null default 0,
    total_damage_done int not null default 0,
    total_damage_taken int not null default 0,
    ability_used_count int not null default 0,
    created_at timestamptz not null default now()
);
```

| Поле               | Назначение                               |
| ------------------ | ---------------------------------------- |
| id                 | Идентификатор боевой единицы             |
| battle_id          | Бой                                      |
| player_id          | Владелец персонажа                       |
| player_crop_id     | Конкретная культура игрока               |
| crop_type_id       | Тип культуры                             |
| side               | Сторона боя                              |
| slot               | Позиция персонажа                        |
| start_hp           | Здоровье на старте боя                   |
| end_hp             | Здоровье после боя                       |
| total_damage_done  | Сколько урона нанес                      |
| total_damage_taken | Сколько урона получил                    |
| ability_used_count | Сколько раз использовал суперспособность |
| created_at         | Дата создания записи                     |

## side

```text
left
right
```

# 11. battle_results

## Назначение

Хранит итог боя и начисленные награды.

```sql
create table battle_results (
    battle_id uuid primary key references battles(id),
    winner_player_id uuid not null references players(id),
    loser_player_id uuid not null references players(id),
    winner_remaining_hp int not null,
    loser_remaining_hp int not null default 0,
    hp_delta int not null,
    seeds_reward int not null default 0,
    chest_reward_code varchar(50),
    rating_delta int not null default 0,
    created_at timestamptz not null default now()
);
```

| Поле                | Назначение                           |
| ------------------- | ------------------------------------ |
| battle_id           | Бой                                  |
| winner_player_id    | Победитель                           |
| loser_player_id     | Проигравший                          |
| winner_remaining_hp | Суммарное оставшееся HP победителя   |
| loser_remaining_hp  | Суммарное оставшееся HP проигравшего |
| hp_delta            | Разница HP между командами           |
| seeds_reward        | Награда семечками                    |
| chest_reward_code   | Код выданного сундука                |
| rating_delta        | Изменение рейтинга                   |
| created_at          | Дата сохранения результата           |

# Redis для активного боя

PostgreSQL хранит постоянные данные и историю.

Redis хранит активный бой.

## Пример ключа

```text
battle:{battleId}:state
```

## Пример состояния

```json
{
  "battleId": "b1",
  "status": "in_progress",
  "leftPlayerId": "p1",
  "rightPlayerId": "p2",
  "leftTeam": [
    {
      "unitId": "u1",
      "cropCode": "tomato",
      "hp": 100,
      "targetUnitId": "u5",
      "abilityReady": true,
      "abilityCooldownMs": 0
    }
  ],
  "rightTeam": [
    {
      "unitId": "u5",
      "cropCode": "pepper",
      "hp": 80,
      "targetUnitId": "u1",
      "abilityReady": false,
      "abilityCooldownMs": 5000
    }
  ]
}
```

## Почему Redis

Во время боя состояние меняется часто:

* HP;
* цель атаки;
* готовность способности;
* cooldown;
* статус персонажа.

Писать каждое изменение в PostgreSQL неэффективно.

После завершения боя итог сохраняется в:

```text
battles
battle_units
battle_results
```

# Минимальная логика MVP

1. Игрок входит через Telegram.
2. Создается запись в players.
3. Создается player_currencies.
4. Создается gardens.
5. Создаются 3 стартовые garden_plots.
6. Игрок получает стартовые культуры в player_crops.
7. Игрок формирует команду в player_team.
8. Создается бой в battles.
9. Активный бой хранится в Redis.
10. После завершения боя результат сохраняется в battle_units и battle_results.
